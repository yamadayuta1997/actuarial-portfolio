DAY 1 · IBNR
チェーンラダー法でIBNRを推計した
損保アクチュアリーの中核業務の一つであるIBNR（発生済み未報告事故）の推計を実装。開発三角形の概念を理解し、発展ファクターを計算してNaNの部分を埋めていく流れを手で実装することで、机上の知識との差が明確になった。
# 発展ファクターの計算 for i in range(len(cols) - 1): valid = triangle[[cols[i], cols[i+1]]].dropna() factor = valid[cols[i+1]].sum() / valid[cols[i]].sum() factors.append(factor) # 12→24ヶ月: 1.5000 24→36ヶ月: 1.1212 36→48ヶ月: 1.0588
IBNR合計 3,279 ／ 2021〜2024年度の未報告分を推計
🤖
AI活用ポイント：「chainladder パッケージを使えば1行で書けるが、あえて手実装した」とAIが提案してくれた。ブラックボックスにしないという意識を持てた。

DAY 1 · GLM
GLMで料率ファクターを分析した
年齢・車種・地域を説明変数としたポアソン回帰GLMを実装。係数をexp変換して「何倍リスクが高いか」と解釈する方法を理解した。単純な集計と違い、複数のリスクファクターを同時に制御できることがGLMの強みだと体感した。
import statsmodels.api as sm import numpy as np model = sm.GLM(y, X, family=sm.families.Poisson()) result = model.fit() # exp変換で料率ファクターとして解釈 vehicle_factor = np.exp(result.params['vehicle_large']) # → 1.45 : 大型車は小型車の1.45倍の事故率
大型車ファクター 1.45× ／ 年齢ファクター 0.978×/歳
🤖
つまずいた点：係数をそのまま読んで「年齢が1歳上がると102%変化」という誤解をした。AIに指摘され、ポアソン回帰ではlog linkを使うためexp変換が必要だと理解した。
GLM ポアソン回帰 料率ファクター statsmodels
DAY 2 · リスク定量化
モンテカルロ法とBootstrap法でリスクを数値化した
1万回のシミュレーションで損害分布を推計し、99%VaRと異常危険準備金を算出。さらにBootstrap法でVaR推計値の信頼区間を計算した。「点推定だけでなく誤差も報告する」というアクチュアリーとしての姿勢を学んだ。
# モンテカルロ: 損害分布のシミュレーション sim_losses = [] for _ in range(10_000): freq = np.random.poisson(claim_freq * 1000) sev = np.random.exponential(avg_severity, freq) sim_losses.append(sev.sum()) var_99 = np.percentile(sim_losses, 99) reserve = var_99 - np.mean(sim_losses)
99% VaR ¥37,665,130 ／ 異常危険準備金 ¥12,645,471 ／ 安全割増率 50.5%
🤖
AI活用ポイント：Bootstrap法の必要性が最初わからなかった。AIに「VaR自体の推計誤差を経営陣に伝えるための手法」と説明してもらい、実務での意義が腹落ちした。
モンテカルロ法 Bootstrap法 VaR 異常危険準備金 信頼区間

# 最終アウトプット print("【料率改定提言】") print(f" 40代料率: {np.exp(result.params['age_group_40s']):.3f}倍 → 引き下げ検討") print(f" 異常危険準備金: {reserve:,.0f}円 の積み立てを推奨") # → 40代料率: 0.665倍 → 引き下げ検討 # → 異常危険準備金: 7,036,539円 の積み立てを推奨
